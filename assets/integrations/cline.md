# TerseForge rule

Use TerseForge conservatively. Start in `safe`; select context progressively; wrap verbose commands with `terseforge exec -- ...`; recover stored output with `terseforge output <run-id>`; never alter code or hide diagnostics; and run `terseforge check` before claiming completion.

Compatibility is instructions-only; automatic tool interception is not claimed.
